####HTTPS握手过程
```
1.客户端使用HTTPS的url访问服务端 要求建立sls
2.服务端收到请求后吧网站的证书和公钥发给客户端
3.客户端收到证书后检查证书的合法机构和有效期如果没有问题就随机产生一个秘钥
4.客户端利用公钥将回话秘钥加密并传给服务端服务端解密出回话秘钥
5.之后服务器和客户端使用秘钥进行传输数据
```
####客户端如何验证证书的合法性
```
1.校验证书的颁发机构是否受客户端信任
2.证书是否被吊销
3.校验证书是否在有效期内
4.校验对方是否存在证书的私钥 判断证书的网站域名是否和颁发证书的域名一致
```
####对HTTP2的了解
```
在HTTP1.0中每次请求都会建立一个连接也就是三次握手和四次挥手这个过程占用了一定时间虽然开启了Keep-Alive解决了多次连接问题
但依然有两个效率上的问题
1.串行文件传输
2.连接数过多
HTTP2.0的多路复用主要是为了解决上边这两个问题
HTTP2采用二进制格式传输取代了1.0的文本格式解析更加高效
所有相同域名的请求都会在同一个tcp连接并发完成
   同级域名下所有通信在单个连接上完成消除了多个tcp连接带来的延时和内存消耗
   可以并行进行请求和相应之间互不影响
```
