模块化的演进过程
```
1.全局func模式，吧不同的模块封装到不同的function进行调用
缺点：污染全局命名空间 容易造成命名冲突
2.namespace模式，简单对象封装
缺点：数据不安全，外部可以直接修改对象内的变量
3.IIFE模式 匿名函数自调用(闭包)
缺点：模块间的依赖比较麻烦
```
2.使用模块化的好处
```
1.避免命名冲突
2.更好的分离，按需加载
3.更高的复用性，维护性
```
commonjs规范
```
node采用commonjs，一个文件就是一个模块，每个模块都有自己的作用域，模块内部的变量
函数 类 都是私有的采用的是同步加载如果放在浏览器端需要提前编译
特点：
1.不会污染全局作用域
2.模块多次加载会进行缓存
3.模块加载的顺序按在代码中出现的顺序
4.commonjs拿到的是值的拷贝 拿到的数据与原来模块的数据没有任何关系
```
ES6模块化
```
ES6模块化的思想是设计尽量静态化,编译的时候就能确定模块的依赖关系commonjs只能在运行时
确定这些关系
es6模块化与commonjs模块化差异
1.commonjs输出值得拷贝，es6输出的是接口或者值得引用
2.commonjs是运行时加载,es6是编译时输出接口
```
